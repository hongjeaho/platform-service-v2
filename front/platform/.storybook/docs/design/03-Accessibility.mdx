# 접근성 가이드

## WCAG AA 준수

모든 컴포넌트는 WCAG 2.1 AA 기준을 준수합니다.

### 색상 대비

- **일반 텍스트**: 최소 4.5:1 대비율
- **큰 텍스트 (18px+)**: 최소 3:1 대비율
- **UI 컴포넌트**: 최소 3:1 대비율

### 키보드 접근성

모든 인터랙티브 요소는 키보드로 접근 가능합니다:

- `Tab`: 다음 요소로 이동
- `Shift + Tab`: 이전 요소로 이동
- `Enter` / `Space`: 버튼 활성화
- `Arrow Keys`: 라디오/셀렉트 네비게이션

### 스크린 리더 지원

- 모든 폼 요소에 `label` 필수
- 버튼에 명확한 텍스트 또는 `aria-label`
- 에러 메시지는 `aria-describedby`로 연결

## Storybook Accessibility 탭 사용법

1. 원하는 컴포넌트 스토리 선택
2. 하단 패널에서 "Accessibility" 탭 클릭
3. 자동 검사 결과 확인
4. 위반 사항이 있다면 상세 설명 확인
5. 수정 후 재검사

### 주요 검사 항목

- **color-contrast**: 색상 대비 검사
- **label**: 폼 요소 라벨 검사
- **button-name**: 버튼 텍스트 검사
- **link-name**: 링크 텍스트 검사
- **aria-***: ARIA 속성 검사

## 접근성 Best Practices

### 폼 컴포넌트

```typescript
// ✅ 올바른 예: label과 연결
<label htmlFor="email">이메일</label>
<input id="email" type="email" />

// ❌ 잘못된 예: label 없음
<input type="email" placeholder="이메일" />
```

### 버튼 컴포넌트

```typescript
// ✅ 올바른 예: 명확한 텍스트
<button>저장</button>

// ❌ 잘못된 예: 아이콘만 있음
<button><SaveIcon /></button>

// ✅ 올바른 예: aria-label 사용
<button aria-label="저장"><SaveIcon /></button>
```

### 색상 대비

```typescript
// ✅ 올바른 예: 4.5:1 이상
<p className="text-foreground">일반 텍스트</p>

// ❌ 잘못된 예: 낮은 대비
<p className="text-gray-300">읽기 어려운 텍스트</p>
```

## 테스트 방법

1. **자동 테스트**: Storybook Accessibility 탭
2. **키보드 테스트**: 모든 기능을 키보드만으로 테스트
3. **스크린 리더 테스트**: NVDA (Windows) 또는 VoiceOver (Mac) 사용
4. **모바일 접근성**: TalkBack (Android) 또는 VoiceOver (iOS) 테스트

## 정부 플랫폼 접근성 요구사항

정부 웹사이트는 한국형 웹 콘텐츠 접근성 지침(KWCAG) 2.1을 준수해야 합니다:

- **인식의 용이성**: 텍스트 대체, 색상 대비
- **운용의 용이성**: 키보드 접근성, 충분한 시간 제공
- **이해의 용이성**: 명확한 지시사항, 일관성
- **견고성**: 웹 표준 준수, 마크업 오류 없음

모든 컴포넌트는 이 기준을 충족하도록 설계되었습니다.
