# 비로그인 사용자 비밀번호 변경 고도화 정의서 (확정)

## 기능 개요
비로그인 사용자가 이메일로 본인 확인(OTP 인증) 후 비밀번호를 변경한다.
기존 "현재 비밀번호 입력" 방식을 "OTP 이메일 인증" 방식으로 완전히 대체한다.

## 기능 요구사항
- 이메일로 본인임을 확인하고 비밀번호를 변경할 수 있다.
- 이메일 인증은 OTP(One-Time Password) 개념을 사용한다.
- OTP의 유효시간은 30분이다.
- OTP 재발송 간격은 10분이다.
- 이메일 발송은 비동기로 처리하며, 응답 시간은 3초 이내여야 한다.

## 제약사항
- 공통 이메일 발송 기능(`PasswordChangeEmailSender`)을 사용한다.
- Redis 캐시를 사용하여 OTP를 저장/검증한다. (신규 구현)
- 기존 `/api/public/users/change-password` API를 OTP 방식으로 완전히 대체한다. (Breaking Change)

## 사용자 시나리오

### 시나리오 1: OTP 발송
1. 사용자가 이메일을 입력하여 OTP 발송을 요청한다.
2. 시스템은 해당 이메일이 등록된 사용자인지 확인한다.
   - 미등록 이메일이면 에러를 반환한다.
3. 시스템은 6자리 OTP 코드를 생성한다.
4. OTP를 Redis에 저장한다 (TTL: 30분).
5. `PasswordChangeEmailSender`를 통해 이메일로 OTP를 발송한다 (비동기).
6. 발송 결과를 반환한다.

### 시나리오 2: 비밀번호 변경
1. 사용자가 이메일, OTP, 새 비밀번호를 입력하여 비밀번호 변경을 요청한다.
2. 시스템은 Redis에서 해당 이메일의 OTP를 조회한다.
   - OTP가 없거나 만료(30분 경과)했으면 에러를 반환한다.
3. 시스템은 OTP가 일치하는지 확인한다.
   - 불일치하면 에러를 반환한다.
4. 시스템은 새 비밀번호가 현재 비밀번호와 다른지 확인한다.
   - 동일하면 에러를 반환한다.
5. 비밀번호를 변경하고 Redis에서 OTP를 삭제한다.
6. 변경 결과를 반환한다.

## 경계 조건
- **OTP 재발송**: 동일 이메일로 10분 미경과 시 재발송 불가
- **OTP 만료**: 발송 후 30분 경과 시 자동 만료
- **OTP 검증 횟수**: 제한 없음 (만료까지 재시도 가능)

## 에러 처리

| 에러 케이스 | HTTP 상태코드 | 메시지 |
|------------|--------------|--------|
| OTP 만료 (30분 경과) | 400 | OTP가 만료되었습니다. 다시 발송해주세요. |
| OTP 불일치 | 400 | OTP가 일치하지 않습니다. |
| 이메일 미등록 사용자 | 400 | 해당 이메일로 등록된 사용자가 없습니다. |
| 재발송 간격 미달 (10분 미경과) | 409 | OTP는 10분마다 재발송할 수 있습니다. |
| 등록된 비밀번호와 동일한 비밀번호로 변경 시도 | 409 | 현재 비밀번호와 동일한 비밀번호로 변경할 수 없습니다. |

## 기술 제약
- **접근 권한**: 인증 불필요 (`/api/public/users/**`, `PublicUsersController`)
- **비동기 처리**: 이메일 발송은 `@Async`로 비동기 처리
- **응답 시간**: 3초 이내
- **OTP 생성**: 6자리 숫자
- **Redis TTL**: 30분
- **재발송 제한**: 10분

## 용어 정의

| 용어 | 설명 | 코드 네이밍 |
|------|------|------------|
| OTP | One-Time Password, 6자리 숫자, 30분 유효 | `otpCode`, `otp` |
| 이메일 발송 | PasswordChangeEmailSender.send() 사용 | `PasswordChangeEmailSender` |
| Redis 캐시 | OTP 저장용, TTL 30분, 재발송 10분 제한 | `RedisTemplate`, `@Cacheable` |
| 비밀번호 변경 | UsersRepository.updatePassword() 사용 | `changePasswordBeforeLogin` |
| 사용자 | UsersEntity, 이메일로 식별 | `UsersEntity`, `userEmail` |
