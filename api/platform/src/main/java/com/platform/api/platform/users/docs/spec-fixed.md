# 회원 등록 확정 정의서

## 기능 개요
플랫폼 사이트에 회원 등록하는 공개 REST API를 구현한다.
인증 없이 최소한의 정보(아이디, 이름, 이메일, 비밀번호)만으로 가입할 수 있으며,
등록된 회원은 자동으로 `USER` 권한을 부여받는다.

## 용어 정의

| 용어 | 설명 | 코드 네이밍 |
|------|------|-----------|
| 회원 | 플랫폼에 등록된 사용자 | `Users` / `UsersEntity` |
| 회원 등록 | 신규 회원을 시스템에 등록하는 행위 | `signup` |
| 사용자 아이디 | 로그인에 사용하는 고유 식별자 | `userId` |
| 사용자 이름 | 회원의 표시 이름 | `userName` |
| 사용자 이메일 | 연락처 이메일 주소 (필수 입력, unique) | `userEmail` |
| 비밀번호 | 로그인 인증에 사용하는 문자열 (BCrypt 해시 저장) | `password` |
| USER 권한 | 일반 회원에게 부여되는 기본 권한 | `"USER"` (roles.role_name) |

## 기능 요구사항

### 필수 입력
- `userId`: 사용자 아이디 (필수, 중복 불허)
- `userName`: 사용자 이름 (필수)
- `password`: 비밀번호 (필수, BCrypt 인코딩 후 저장)
- `userEmail`: 사용자 이메일 (필수, 중복 불허)

### 등록 처리
- 아이디 중복 시 409 에러 반환
- 이메일 중복 시 409 에러 반환
- 비밀번호는 BCryptPasswordEncoder로 인코딩하여 저장
- 등록 완료 후 자동으로 USER 권한 부여 (roles 테이블의 role_name='USER' row와 연결)
- 공개 API: 인증 없이 접근 가능

## 제약사항

- 테이블 변경 없음 (DDL 변경 금지)
- `created_by`: 공개 API라 인증 사용자가 없으므로 `0L` (시스템 계정) 사용
- `user_email`은 DB nullable이나 비즈니스 레이어에서 필수 + unique 강제

## 에러 처리 정책

| 상황 | 예외 | HTTP |
|------|------|------|
| 아이디 중복 | `IllegalStateException` | 409 |
| 이메일 중복 | `IllegalStateException` | 409 |
| 필수 필드 누락 | `@Valid` 자동 처리 | 400 |
